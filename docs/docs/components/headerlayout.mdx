# HeaderLayout

The `HeaderLayout` component provides a full flex-based page layout system. Unlike `AppShell` (which is viewport-bound), `HeaderLayout` is designed for **scrolling document pages**.

It supports sticky headers, collapsible icon-only sidebars, and responsive stacking.

## Attributes

| Attribute | Target | Value | Description |
|---|---|---|---|
| `data-header-layout` | Root `<div>` | (present) | Activates the flex column page layout |
| `data-sticky` | `<header>` | (present) | Pins header to top of viewport |
| `data-body` | `<div>` | (required) | Wrapper for sidebar + main |
| `data-collapsed` | `<aside>` | (present) | Shrinks sidebar to `4.5rem` icon-only mode |
| `data-label` | Inside `<aside>` items | (present) | Text hidden when sidebar is collapsed |

---

## CSS Variables

| CSS Variable | Default | Description |
|---|---|---|
| `--ix-sidebar-width` | `16rem` | Expanded sidebar width — override in your stylesheet |

---

## Responsive Behavior

On screens **≤ 768px**, `[data-body]` stacks vertically — sidebar moves above `<main>` with a bottom border replacing the right border.
